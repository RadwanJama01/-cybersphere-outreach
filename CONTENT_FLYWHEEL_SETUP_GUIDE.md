# Cybersphere Content Flywheel — Implementation Guide

**Purpose:** Step-by-step instructions to build the full Aura Farming Engine in Notion, Tally, Make, Buffer, and Slack.

## Step 3 — Connect Tally → Notion via Make.com

### 3.1 Create a Make.com scenario: "Brain Dump → Notion"

**Trigger:** Tally — "New Response"
**Module chain:**

1. **Tally trigger** → receives form response
2. **Router** → split by response type:

**Route A — Credentials:**
3. Notion → "Create Database Item" in Credentials table
   - Map: Institution = form field, Role = form field, Key Work = form field, etc.
   - Set Person = dropdown value from form

**Route B — Client Projects:**
4. Notion → "Create Database Item" in Client Projects table
   - Map all fields from the client section of the form
   - Set Content Status = "Not Started"

**Route C — Associations:**
5. Notion → "Create Database Item" in Associations table
   - Parse the associations text field
   - Create one row per association mentioned

**Route D — Narrative Arcs:**
6. Notion → "Create Database Item" in Narrative Arcs table
   - Map arc title + core story

### 3.2 Create a Make.com scenario: "Auto-Generate Angles"

**Trigger:** Notion — "Database Item Created" in Client Projects
**Module chain:**

1. Notion trigger → new client project added
2. HTTP module → Call Claude API with this prompt:

```
Given this client project:
Client: {{Client}}
Industry: {{Industry}}
Problem: {{Problem Before}}
Solution: {{Solution Built}}
Outcome: {{Outcome}}

Generate exactly 10 content angles, one per line:
1. Process angle — "Here's exactly how we built..."
2. Outcome angle — "This AI system generated..."
3. Mistake angle — "We almost screwed this up..."
4. Decision angle — "The one call we made..."
5. Credential angle — "What [credential] taught me..."
6. Contrarian angle — "Everyone said [X]..."
7. Behind-the-scenes angle — "What the client never saw..."
8. Lesson angle — "3 things I'd do differently..."
9. Industry angle — "Why [industry] is perfect for AI..."
10. Founder journey angle — "From Stanford to..."

Each angle should be a specific, compelling one-liner that could be a post hook.
```

3. Notion → Update the Client Projects item:
   - Set "Angles Generated" = 10
   - Append angles to a text field or create 10 Content Queue items

### 3.3 Create a Make.com scenario: "Generate Content Assets"

**Trigger:** Manual or scheduled (run after angles are generated)
**For each angle x format combination:**

1. Read the client project + angle from Notion
2. Call Claude API with the generation prompt structure from the spec:

```
CONTEXT:
- Case study: [from Notion]
- Angle: [from angle list]
- Format: [LinkedIn post / Medium article / etc.]
- Brand voice: Direct, credible, founder-first. Never corporate.
  Reference Stanford/MIT/Databricks when relevant.
  Speak from lived experience. No fluff, no AI buzzwords.
  Always end with a subtle pull toward booking a call or following.

GENERATE:
[full asset in the specified format]

ALSO OUTPUT:
- Hook type tag
- Objection addressed
- Cachet asset used
- CTA type
- Recommended platform
- Recommended posting time
```

3. Create a new item in the Content Queue table with all fields populated
4. Set Status = "Draft"

**Schedule:** Every day at 7:00 AM ET

**Module chain:**

1. **HTTP module** → Call Perplexity API:
   - Query 1: "top AI news today"
   - Query 2: "AI automation news for small businesses"
   - Query 3: "AI voice agent news"

   - Angle = "Trending News"
   - Status = "Pending Review"
   - Scheduled Date = today

   "Today's AI news angles are ready — 3 posts in the queue. Approve or edit before 10am: [Notion link]"

## Step 7 — Set Up the Posting Agent (Buffer)

### 7.1 Connect platforms to Buffer
- LinkedIn (Neetish's personal profile)
- Instagram (Cybersphere business account)
- Threads
- Twitter/X

### 7.2 For platforms Buffer doesn't support:
- Medium → Use Medium API (POST https://api.medium.com/v1/users/{userId}/posts)
- Substack → Manual or RSS-to-email via Zapier
- Reddit → Use Reddit API (POST /api/submit)
- Quora → Manual (no reliable API)
- YouTube → Manual upload with Claude-generated script + description

### 7.3 Create a Make.com scenario: "Approved → Buffer"

**Trigger:** Notion — "Database Item Updated" in Content Queue (when Status changes to "Approved")

**Module chain:**

1. Notion trigger → Content Queue item status = "Approved"
2. Router by Platform:
   - LinkedIn → Buffer API: Schedule post
   - Instagram → Buffer API: Schedule post
   - Threads → Buffer API: Schedule post
   - Medium → HTTP module: Medium API create post
   - Reddit → HTTP module: Reddit API submit
3. Notion → Update Content Queue item:
   - Status = "Scheduled"
   - Add Post URL when published

### 7.4 Posting cadence enforcement
- LinkedIn: Max 1x/day, schedule for 8-9 AM ET
- Medium: Max 3x/week, schedule for 10 AM ET
- Substack: Max 2x/week, schedule for 8 AM ET
- Instagram: Max 2x/day, schedule for 12 PM and 6 PM ET
- Threads: Max 3x/day, schedule for 9 AM, 1 PM, 6 PM ET
- YouTube: Max 1x/week, schedule for Saturday 10 AM ET
- Reddit: Max 3x/week, schedule for 11 AM ET (r/smallbusiness, r/artificial, r/entrepreneur)
- Quora: Max 2x/week, schedule for 2 PM ET

## Step 8 — Set Up Performance Tracking (Make.com)

### 8.1 Create a Make.com scenario: "Daily Performance Pull"

**Schedule:** Every day at 11:00 PM ET

**Module chain:**

1. **Notion** → Get all Content Queue items where Status = "Published" and Published Date = last 7 days
2. For each item, by platform:
   - LinkedIn → LinkedIn API: Get post analytics
   - Instagram → Instagram Graph API: Get media insights
   - Medium → Medium API or manual
   - Buffer → Buffer Analytics API
3. **Notion** → Update Content Queue items with:
   - Views, Likes, Comments, Shares, Saves
4. **Check winner formula** → If Saves + Shares > threshold:
   - Tag as "Winner"
   - Trigger remix generation (see Step 9)

### 8.2 Manual tracking fallback (V1)
If API setup is too complex for V1, create a Notion view:
- Filter: Status = "Published", last 7 days
- Sort by: Published Date descending
- Neetish manually enters metrics from each platform's analytics once daily (~5 min)

## Step 9 — Set Up Winner Detection + Auto-Remix (Make.com)

### 9.1 Create a Make.com scenario: "Winner → Remix"

**Trigger:** Notion — "Database Item Updated" in Content Queue (when Winner? = true)

**Module chain:**

1. Notion trigger → Content Queue item flagged as Winner
2. Read the winning post's: hook type, angle, format, cachet used, full content
3. **Claude API** → Generate remixes:

```
This post was a top performer:
Title: [title]
Platform: [platform]
Hook type: [hook type]
Angle: [angle]
Content: [full content]

Generate:
1. 5 new hooks (same body, different opening line)
2. 3 cross-platform adaptations:
   - If original was LinkedIn → generate Threads thread + Instagram carousel + Medium article outline
   - If original was Instagram → generate LinkedIn post + Threads thread + Reddit post
3. 1 "meta-post" (Theory of Reflecting Mirrors): Neetish reflecting on why this post performed well

For each output, include: hook type tag, recommended platform, recommended posting time.
```

4. **Notion** → Create new Content Queue items for each remix
   - Set Priority = "Winner Remix"
   - Set Remix Of = relation to original post
   - Set Status = "Pending Review"
5. **Slack** → Notify: "Winner detected! [post title] on [platform]. 9 remixes queued — review in Notion: [link]"

## Step 10 — Set Up the Daily Digest Bot (Make.com → Slack)

### 10.1 Create a Make.com scenario: "Daily Content Digest"

**Schedule:** Every day at 8:00 AM ET

**Module chain:**

1. **Notion** → Get Content Queue items where Published Date = yesterday
2. **Notion** → Get Content Queue items sorted by (Saves + Shares) descending, last 7 days, limit 3 (Top performers)
3. **Notion** → Get Content Queue items sorted by (Saves + Shares) ascending, last 7 days, limit 3 (Bottom performers)
4. **Notion** → Count items by Status: Approved, Draft, Pending Review
5. **Claude API** → Format digest:

```
Format this data into the daily digest:

Posted yesterday: [list]
Top 3 this week: [list with why they won]
Bottom 3 this week: [list with why they underperformed]
Queue status: [counts]

Generate 3 specific action recommendations for today.
Format as a clean Slack message with emoji headers.
```

6. **Slack webhook** → Send to #content-engine channel

- Go to api.slack.com/apps → Create New App
